CSE 340 Compiler Project

This is the first project completed as part of my CSE 340 Principles of Programming Language class in the spring semester of 2020. The project required us to create a parser for a pseudo programming language. The program takes mathematical polynomials as inputs. A full description of the project can be found here.

About the project

This script utilizes linked list data structures to parse and execute input files. The project is a compiler which utilizes its own programming language of sorts, and scans input "code" for syntax errors and compiles it into executable memory.

First the program reads in an input file and verifies the syntax and semantics are correct. If it completes successfully, the program evaluates polynomials and outputs the results. Below is an example of an input:

POLY F = x^2 + 1;
POLY G = x + 1;
START
INPUT X;
INPUT Y;
F(X);
F(Y);
1 2 3 18 19

At the beginning of the input two polynomials F, and G are declared. In the start section, variables are loaded and the polynomials are executed. There is an input buffer containing 1 2 3 18 19 however not all of these values are used. First INPUT X will assign X = 1 and INPUT Y will assign Y = 2. The other 3 values are not used. The next two lines will execute these polynomials with the variables as arguments. The output of this script is 2 3 since F(1) = (1) ^ 2 + 1 = 2 and G(2) = (2) + 1 = 3
