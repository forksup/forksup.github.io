About Bidirectional Search

In this project we implemented Bidirectional search as described in the paper, "Bidirectional Search That Is Guaranteed to Meet in the Middle". This algorithm works be running two searches. One from the goal position and another from the start position. These two searches run until they meet in the middle and thus a solution is found. There are two varients of this algorithm, Bidirectional Search (mm) and Brute Force Bidirectional Search (mm0). Brute force works similarly to BFS and does not use any type of search strategy. Our Findings

Below is a paper demonstrating all of our findings. We compared this search technique with several others including Breadth First Search, Depth First Search, Uniform Cost Search, and A*. We used the number of nodes expanded as the main metric for this comparison. This value reflects the amount of resources and time utilzied by the script. We found Bidirectional search to be optimal over traditional search algorithms using this metric.
